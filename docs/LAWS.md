# Product laws

| ID | Law |
|----|-----|
| **M1** | **Own pack** — each seat has one canonical memory root; write only there by default |
| **M2** | **No silent cross-write** — never edit another seat’s spine/journal/worklog |
| **M3** | **House search** — registered packs + shared stores are readable via `/hive` |
| **M4** | **Attribution required** — foreign spans: `owner`, `when`, `kind`, `status`, `not_me` |
| **M5** | **Searcher owns assembly** — the active seat runs hive search; no silent PTY inject |
| **M6** | **Budget ceiling** — ~30–50k tokens per load/assembly; report drops |
| **M7** | **Promote explicit** — copy a law into own spine only on deliberate ritual |
| **M8** | **Kinds stay separate** — coding-pack ≠ companion-mind ≠ board ≠ ledger ≠ code-graph |
| **M9** | **No transcript-as-continuity** — distill, don’t replay 300k chats |
| **M10** | **Secrets never** — no API keys, auth files, or credentials in packs |
| **M11** | **Scar honesty** — struggles and faceplants are first-class retrieval targets |
| **M12** | **Audit posture** — when hive-loading foreign work, reason as reviewer of *their* trail |
| **M13** | **Switchable seats** — registry is path + owner metadata; no hard-coded single vendor |
| **M14** | **Human gates** — promote-to-house-law and destructive pack ops need human intent |
| **M15** | **`/hive` additive** — wrap/recall stay own-pack only; never auto-merge hive into blank recall |
| **M16** | **Wrap hive hygiene** — `## From the hive` for peer evidence; `## Recall` = this seat only |

## One-line

> You own your memories. When you want the hive, ask for it. You never confuse the two.
